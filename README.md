# IVSSF-RL for Dynamic Obstacle Avoidance

This repository provides the source code, simulation environments, and learned fuzzy rule bases for the paper:

**Interpretable Variable-Structure Sparse Fuzzy Reinforcement Learning for Dynamic Obstacle Avoidance**

The proposed IVSSF-RL method represents the robot navigation policy using a sparse fuzzy rule base instead of a conventional black-box neural actor. During training, the rule base is expanded when the current state is insufficiently covered and aggregated when redundant fuzzy sets or rules appear. The method is evaluated on TurtleBot3-based dynamic pedestrian avoidance tasks in Gazebo.

## Repository Structure

```text
.
├── IVSSF_RL/
│   └── Source code of the proposed IVSSF-RL algorithm
│
├── rule_base/
│   └── Original learned fuzzy rules used for interpretability analysis
│
└── turtlebot3_dynamic_pedestrians/
    └── Gazebo simulation environments and launch files
````

## Folder Description

### `IVSSF_RL`

The `IVSSF_RL` folder contains the implementation of the proposed algorithm, including:

* Sparse fuzzy actor
* Rule-base expansion
* Rule-base aggregation
* Antecedent generalization
* Off-policy actor-critic learning
* Training and evaluation scripts

This folder corresponds to the main source code of the proposed IVSSF-RL method.

### `rule_base`

The `rule_base` folder contains the original learned fuzzy rules obtained from the trained IVSSF-RL agent.

These rules are used in the interpretability analysis section of the paper. The displayed interpretable rule subset is extracted from the learned rule base according to the activation patterns of representative states.

### `turtlebot3_dynamic_pedestrians`

The `turtlebot3_dynamic_pedestrians` folder contains the Gazebo simulation environments used in the paper.

The simulation environments are based on the TurtleBot3 mobile robot and dynamic pedestrian models.

## Simulation Environments

Three Gazebo simulation scenarios are used in the paper: Crosswalk, 8-pedestrian, and Generalization.

### 1. Crosswalk Scenario

The Crosswalk scenario contains intersecting pedestrian flows and is used to evaluate the basic dynamic obstacle avoidance capability of the learned policy.

It is used for both training and in-domain evaluation.

Launch command:

```bash
roslaunch turtlebot3_dynamic_pedestrians social_teacher_cross_dual_lane.launch
```

### 2. 8-Pedestrian Scenario

The 8-pedestrian scenario contains a denser pedestrian flow and provides a more challenging environment for testing policy robustness in crowded dynamic scenes.

It is used for both training and in-domain evaluation.

Launch command:

```bash
roslaunch turtlebot3_dynamic_pedestrians social_train_hybrid_8ped.launch
```

### 3. Generalization Scenario

The Generalization scenario contains a different spatial layout and pedestrian arrangement from the training environments.

It is used only for zero-shot evaluation of the agent trained in the 8-pedestrian scenario. No additional fine-tuning is performed in this environment.

Launch command:

```bash
roslaunch turtlebot3_dynamic_pedestrians social_teacher_wall_8ped_opposite.launch
```

## Environment Usage

Before running the training or evaluation scripts, launch the corresponding Gazebo environment.

For the Crosswalk scenario:

```bash
roslaunch turtlebot3_dynamic_pedestrians social_teacher_cross_dual_lane.launch
```

For the 8-pedestrian scenario:

```bash
roslaunch turtlebot3_dynamic_pedestrians social_train_hybrid_8ped.launch
```

For the Generalization scenario:

```bash
roslaunch turtlebot3_dynamic_pedestrians social_teacher_wall_8ped_opposite.launch
```

After the simulation environment is launched, the training or evaluation scripts in `IVSSF_RL` can be executed.

## Experimental Protocol

The Crosswalk and 8-pedestrian scenarios are used for training and in-domain evaluation.

The Generalization scenario is used only to evaluate the transferability of the agent trained in the 8-pedestrian scenario.

The evaluation results reported in the paper include:

* Success rate
* Average navigation steps
* Minimum distance to humans
* Training time
* Learning efficiency
* Final rule-base size

## Interpretability Analysis

The interpretability analysis is based on the learned fuzzy rule base stored in the `rule_base` folder.

The rules shown in the paper are not manually designed. They are extracted from the learned rule base generated by the IVSSF-RL training process.

For representative navigation states, the significantly activated rules are selected and visualized to explain the decision-making behavior of the learned sparse fuzzy policy.
